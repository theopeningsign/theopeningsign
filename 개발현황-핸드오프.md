## 1. 핵심 변경 ①: 이미지 문제 (기존 → 현재)

### 기존 방식 (문제)
- Notion이 주는 이미지 URL은 **약 1시간 뒤 만료되는 S3 서명 URL**.
- 그 URL을 페이지에 그대로 박아 클라이언트에 노출 → 시간이 지나면 깨짐.
- 그걸 가리려고 **"깨지면 새로고침/재시도" 땜빵을 7겹** 쌓아둠 (router.refresh 재시도, 강제 reload, bfcache reload, 5초 failsafe, 재시도 모달, 이미지 중복 프로빙, 폴링 등).
- 결과: 스피너 깜빡임, 화면 움찔, 무한 스피너, 데이터 낭비(같은 사진 여러 번 다운로드 → 페이지 100MB+).

### 현재 방식 (해결)
- **이미지 프록시 라우트** `src/app/api/img/[...slug]/route.ts` 도입.
- 클라이언트엔 **안 죽는 내부 주소** `/api/img/<pageId>/<cover|add>/<index>` 만 노출.
- 이 라우트가 요청 시점에 Notion에서 **싱싱한 S3 URL을 받아 이미지 바이트를 직접 전달(스트리밍)**.
- → 만료 자체가 사용자에게 안 보임. **언제 봐도 안 깨짐.** 재시도 땜빵 전부 제거(`imageReload.ts`, `AutoRefresh.tsx` 삭제).
- 페이지 용량도 대폭 감소(중복 다운로드/리로드 제거).

### ⚠️ 이 부분 주의 (퇴행 금지 / 함정)
- **"깨지면 새로고침/재시도" 로직을 다시 넣지 말 것.** 원인(만료)은 프록시가 이미 막았음. 재시도 추가 = 퇴행.
- 프록시는 **반드시 "바이트 직접 전달(스트리밍)"** 이어야 함.
  - ❌ 307 리다이렉트 방식 → 크롬 **ORB(`ERR_BLOCKED_BY_ORB`)** 로 이미지 차단됨.
  - ❌ `crossOrigin` 붙이기 → 리다이렉트 CORS 에러로 더 악화됨.
  - ✅ 서버가 바이트를 직접 내려주면 브라우저가 외부로 안 나가서 ORB/CORS 자체가 발생 안 함.
- **이미지 최적화(next/image optimization)는 끈 채로 유지** (`unoptimized`). Vercel 무료 "Image Optimization" 한도 때문에 일부러 끈 것. 켜지 말 것.
  - 단, 메인 히어로의 로컬 이미지(`public/main/*`)도 `unoptimized`. (Next16에서 로컬 이미지에 `?쿼리` 붙이면 `images.localPatterns` 에러 → 쿼리 쓰지 말 것)

## 3. 다른 컴퓨터에서 이어받기 (셋업)

1. `git clone https://github.com/theopeningsign/theopeningsign.git` (또는 기존 폴더에서 `git fetch`)
2. `git checkout feature/image-proxy && git pull`
3. `npm install`
4. **프로젝트 루트에 `.env.local` 생성** (git에 없음 — 직접 넣어야 함). 필요한 변수:
   - `NOTION_API_KEY` — Notion 연동
   - `NOTION_DATABASE_ID` — Notion DB
   - `SITE_URL` (로컬은 http://localhost:3000), `SITE_NAME`, `SITE_DESCRIPTION`
   - `REVALIDATE_SECRET` — 캐시 재검증 라우트용
   - `RESEND_API_KEY` — 메일 발송
   - `CONTACT_TO_EMAIL` — 상담 메일 받을 주소
   - 값은 **기존 컴퓨터의 `.env.local`** 또는 **Vercel 환경변수**에서 복사.
5. `npm run dev` → http://localhost:3000

### 개발서버 팁 (윈도우)
- 백그라운드로 띄울 때 세션에 묶이면 자꾸 죽어서, **독립 프로세스로** 띄우는 게 안정적:
  `Start-Process cmd.exe -ArgumentList "/c","npm run dev" -WindowStyle Hidden`
- **`.env.local`을 바꾸면 dev 서버를 재시작**해야 반영됨.

## 5. 주의할 점 / 교훈 요약

- **퇴행 금지**: 이미지 "재시도/강제 새로고침" 로직 다시 넣지 말 것. 원인은 프록시가 막았음.
- **프록시는 스트리밍 전용**: 307 리다이렉트/crossOrigin 쓰면 ORB·CORS로 깨짐.
- **이미지 최적화 OFF 유지**: Vercel 무료 한도 때문. 포트폴리오/Notion 이미지는 `unoptimized`.
- **Next 16 함정**: 로컬 이미지에 쿼리스트링(`?v=` 등) 붙이면 `images.localPatterns` 에러.
- **Vercel 무료 한도**: 신경 쓸 핵심 지표는 **Fast Data Transfer(100GB/월)**. 이미지 최적화 할당량은 안 씀(거의 0 유지). 함수 호출/대역폭은 캐시로 완화됨.
- **비밀키는 git 금지**: `.env.local`(gitignore됨)에만. 배포처는 Vercel 환경변수.
- **`main`/프로덕션은 검증 전까지 보존.** 모든 작업은 `feature/image-proxy`에서.
- **dev 서버**: 독립 프로세스로 띄우기, env 바꾸면 재시작.
